I've implemented "forum tags". That means each forum can have one or more tags to indicate potentially offensive material. Users can set (at registration, and in the User Control Panel) which tags they're ok with seeing.

The tags are currently:
  • Nudity (Nudity, pornography)
  • Gore (Gore, blood, violent death)
  • Pedo (Pedophilic fiction, lolicon)
  • FreeSpeech (May be offensive to some groups)
That last FreeSpeech one is supposed to take care of v/Jews, v/Niggers etc. For new and existing users it defaults to ON, i.e. everyone can see these subs by default. (The others default to OFF i.e. you can only see that material if you opt-in.) I hope it will reduce complaints/attacks from normies and also remind them that it's actually okay to read/write about "sensitive" material - after all it's FreeSpeechTM.

These changes aren't set in stone, they're more to encourage discussion at this stage. Let me know what you think, and obvs if you find any glitches.

I'm not so sure about this "free speech" tag approach...
I mean, if this is supposed to be as close as possible to a Voat emulation, the idea is that everything (within legal bounds) is "free speech" by default and at most moderated by "the community" and/or moderators of a particular subverse. On Voat, that line of "what is seen when you aren't logged in" was drawn at NSFW; that is, NSFW content and subverses were hidden by default and the only way to see them was to register an account and update your settings as desired. In that context, NSFW referred mainly if not exclusively to pornographic/sexual and gore related content.

I may be remembering incorrectly, but I'm pretty sure that subverses like Jews and Niggers and FatPeopleHate (read: "controversial topics/views") were not "NSFW" and thus were not hidden by default. Granted you aren't hiding such subforums by default, but I think we're diverging a bit from Voat here when we use a "Free Speech" tag as a metric for hiding something (albeit not by default). See, back on Voat, if you didn't like the content of a given subverse you had the ability to outright block it from showing on the front/all pages but again you needed an account for this functionality.

To summarize, I think that given the core principals of voat AND your implementation not hiding the subforums in question by default, the "Free Speech" tag is entirely redundant if you were to replace it with the ability to hide subforums of your choice from showing in "all"/"new" via the user control panel. This of course would require some modification of core functionality of phpbb, but I think it would be better in the long term if you can swing that, versus this redundant "free speech" tag.
