Search is a huge issue for Voat simply because of it's size. We simply do not have the indexing structure in place to facilitate this at the time. If Voat's db was small (oh how I long for the old days) we could turn it back on and it would work, but right now a query can significantly impact CPU and locking.

Search is something we had to put off until we had the means to address it correctly (dedicated backend and separate code devoted to it).

Plus, the search we had was elementary and not worth a damn.
